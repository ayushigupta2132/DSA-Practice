LINKED LIST (Refined Notes)
1. What is a Linked List?
A Linked List is a linear data structure used to store a collection of elements (nodes).
Unlike arrays, elements in a linked list are not stored in contiguous memory locations.

2. Array vs Linked List
Array
• Fixed size
• Elements stored in contiguous memory locations
• Fast access using index
• Insertion and deletion are costly

Linked List
• Dynamic size
• Elements stored in non-contiguous memory locations
• No direct indexing
• Insertion and deletion are efficient

3. Structure of a Linked List
Each element of a linked list is called a Node.

A Node contains:

Data – the value to be stored

Pointer – address of the next node

[data | next] -> [data | next] -> [data | NULL]
• The first node is called HEAD
• The last node points to NULL

4. Starting Point of Linked List
• The starting point of a linked list is stored in a pointer called head
• If head == NULL, the list is empty

head -> Node1 -> Node2 -> Node3 -> NULL
5. Memory Representation Example
Values: 1 → 3 → 2 → 5

Memory (not contiguous):

m1: [1 | m2]
m2: [3 | m3]
m3: [2 | m4]
m4: [5 | NULL]
head = m1
tail = m4
6. Applications of Linked List
• Implementation of Stack
• Implementation of Queue
• Browser history (forward / backward navigation)
• Music playlist
• Undo / Redo operations

Example (Browser History):

search <-> typing <-> A2Z <-> article
7. Why Do We Need a User-Defined Data Type?
In linked lists, we need to store:
• Data
• Pointer to next node

Built-in data types cannot do this together, so we create a user-defined data type.

8. Using struct (C-style)
struct Node {
    int data;
    Node* next;
};
Limitations:
• No access control
• No constructors (in C-style usage)
• Less safe and less object-oriented

9. Why Use class Instead of struct?
Advantages of class:
• Supports OOP concepts
• Data can be private (data security)
• Supports constructors & destructors
• Better for large projects

Key Difference:
struct	class
Members are public by default	Members are private by default
Mostly used in C	Used in C++ (OOP)
10. Node Class with Constructor (Recommended)
class Node {
public:
    int data;
    Node* next;

    // Constructor
    Node(int data, Node* next = nullptr) {
        this->data = data;
        this->next = next;
    }
};
Why Constructor?
• Automatically initializes data
• Reduces errors
• Makes code cleaner

11. Creating Nodes
Method 1: Using Pointer (Recommended)
Node* x = new Node(2, nullptr);
Node* y = new Node(3, x);
Method 2: Without Pointer (Not Preferred)
Node x(2, nullptr);
Node* y = &x;
12. Basic Linked List Example
Node* head = new Node(1);
head->next = new Node(3);
head->next->next = new Node(2);
head->next->next->next = new Node(5);
Result:

1 -> 3 -> 2 -> 5 -> NULL
13. Important Points to Remember (Exam Ready)
• Linked list is dynamic
• No random access
• Extra memory required for pointer
• Efficient insertion & deletion
• Traversal is sequential

14. Summary
• Linked List stores data in nodes
• Nodes are connected using pointers
• class is preferred over struct
• Constructors simplify initialization
• Used in real-life applications like browser history

15. Memory Space used
32-bit system:
Each node uses 8 bytes (4 bytes for int data + 4 bytes for next pointer).

64-bit system:
Each node uses 12–16 bytes (4 bytes for int data + 8 bytes for next pointer, extra due to padding).
